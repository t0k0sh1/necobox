# プルリクエスト作成コマンド

このコマンドは、現在のブランチからプルリクエストを作成する際に使用します。

## 実行前の確認事項

1. **変更内容の確認**

   - 変更したファイルを確認する
   - コミット履歴を確認する
   - 変更内容が意図した通りであることを確認する

2. **テストとリンター**

   - すべてのテストが通過することを確認する
   - リンターエラーや型エラーがないことを確認する
   - 不要なコードやコメントを削除する

3. **ブランチの状態**
   - 最新の main ブランチ（またはベースブランチ）を取り込んでいる
   - マージコンフリクトがない
   - コミットが適切に作成されている

## プルリクエスト作成手順

### 1. タイトルの作成

- 変更内容が一目でわかる簡潔なタイトルを作成する
- プレフィックスを使用する（例: `feat:`, `fix:`, `refactor:`, `docs:`, `test:`）
- 日本語で記述する

### 2. 説明文の作成

以下の項目を含める：

- **変更の目的**: なぜこの変更が必要なのか
- **変更内容**: 何を変更したのか（箇条書きで記載）
- **関連 Issue**: 関連する Issue 番号がある場合は参照する
- **動作確認**: スクリーンショットや動作確認の方法がある場合は記載する
- **チェックリスト**: 以下の項目を確認済みであることを示す
  - [ ] コードが正しく動作することを確認した
  - [ ] リンターエラーや型エラーがない
  - [ ] テストがすべて通過する
  - [ ] 不要なコードやコメントを削除した
  - [ ] 関連するドキュメントを更新した（必要に応じて）

### 3. GitHub API を使用した作成

GitHub MCP ツールが利用可能な場合は、以下の情報を収集してから作成する：

- リポジトリのオーナー名
- リポジトリ名
- ベースブランチ（通常は `main` または `master`）
- ヘッドブランチ（現在のブランチ）
- タイトルと説明文

### 4. 手動での作成

GitHub MCP ツールが利用できない場合は、以下の手順で作成する：

1. GitHub のリポジトリページに移動する
2. 「Pull requests」タブを開く
3. 「New pull request」をクリックする
4. ベースブランチとヘッドブランチを選択する
5. タイトルと説明文を入力する
6. 「Create pull request」をクリックする

## 注意事項

- プルリクエストを作成する前に、必ず変更内容をレビューすること
- 大きな変更の場合は、小さな単位に分割して複数のプルリクエストにすることを検討する
- 緊急の修正など、特別な理由がない限り、レビューなしでマージしないこと
